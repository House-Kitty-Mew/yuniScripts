# Minescript Client Scripts for YuniScripts

Custom Python scripts for the [Minescript](https://minescript.net/) Minecraft mod,
designed to work with the YuniScripts server-side engine.

## Scripts

### `sign_item.py` – Item Signing Client
Right-click with an emerald in your off-hand and any item in your main hand
to get it "signed" (renamed with randomized rarity and a certification code).

- **Encryption:** AES-256-GCM using per-player keys
- **Deduplication:** Client-side 2-second window prevents duplicate requests
- **Network:** Sends encrypted UDP packets to the bridge server on port 25571

### `gen-id.py` – Key Generator
Run `\gen-id <username>` in Minecraft chat to generate a new encryption key.
The key is stored locally in `generated_keys.ini` and printed to chat.

### `mc_status_sender.py` – Status Sender
Sends your current biome and dimension to the YuniScripts engine via UDP
every 5 seconds. Used by the MC Status Relay and Nextcloud Status Updater.

## Setup

1. Install the Minescript mod (requires Minecraft 1.21.2+ and Minescript 5.0+)
2. Place the contents of this `minescript/` folder in your `.minecraft/minescript/` directory
3. Configure `sign_item_keys.ini` with player encryption keys (or generate them via `\gen-id`)
4. Optionally uncomment the autorun line in `config.txt` to auto-start `mc_status_sender`

## Configuration Files

All configuration files for minescript scripts now live in the centralized `DATA/` directory at the YuniScripts project root.
Legacy files in the minescript folder are auto-migrated on first load (copy, not move).

| File | Purpose |
|------|---------|
| `DATA/sign_item_config.json` | Cooldown, timeout, and logging settings (legacy: minescript/sign_item_config.json) |
| `DATA/sign_item_keys.ini` | Per-player base64 encryption keys — keep secret! (legacy: minescript/sign_item_keys.ini) |
| `config.txt` | Minescript autorun and Python path config (local only) |
| `generated_keys.ini` | Keys generated by `\gen-id` command (local only) |

## Dependencies

All scripts use either Python standard library or bundled fallbacks:
- `HELPERS/base64_fallback.py` – pure-Python base64 (used if stdlib unavailable)
- `HELPERS/crypto_fallback.py` – pure-Python AES-256-GCM (used if pycryptodome unavailable)

For best performance, install `pycryptodome`:
```
pip install pycryptodome
```
